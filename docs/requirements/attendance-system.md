# 勤怠管理アプリ — 要求仕様書

## 概要

社員約50名の企業向け Web 勤怠管理システム。出退勤打刻、休暇申請、勤怠集計を提供する。

## ユーザーと権限

| ロール | 説明 |
|--------|------|
| 一般社員 | 自身の打刻・履歴閲覧・休暇申請を行う |
| 管理者 | 全社員の勤怠閲覧・休暇承認・有給付与を行う |

- 認証: ID / パスワード

## ユーザーストーリー

### US-1: ログイン

> 社員として、ID とパスワードでログインしたい。自分の権限に応じた画面が表示される。

- 受け入れ基準:
  - 正しい ID/PW でログインできる
  - 誤った ID/PW ではエラーメッセージが表示される
  - ログイン後、一般社員は打刻画面、管理者は管理画面に遷移する

### US-2: 出勤打刻

> 社員として、Web 画面のボタンを押して出勤を記録したい。

- 受け入れ基準:
  - 「出勤」ボタン押下で現在時刻が出勤時刻として記録される
  - 当日すでに出勤打刻済みの場合はエラーになる
  - 打刻後、打刻済みの状態が画面に表示される

### US-3: 退勤打刻

> 社員として、Web 画面のボタンを押して退勤を記録したい。

- 受け入れ基準:
  - 「退勤」ボタン押下で現在時刻が退勤時刻として記録される
  - 出勤打刻なしで退勤はできない
  - 当日すでに退勤打刻済みの場合はエラーになる

### US-4: 自分の勤怠履歴閲覧

> 社員として、自分の過去の打刻履歴を一覧で確認したい。

- 受け入れ基準:
  - 日付・出勤時刻・退勤時刻の一覧が表示される
  - 月単位で絞り込みができる

### US-5: 管理者による全社員の勤怠閲覧

> 管理者として、全社員の勤怠履歴を閲覧したい。

- 受け入れ基準:
  - 社員を選択して勤怠履歴を表示できる
  - 月単位で絞り込みができる

### US-6: 休暇申請

> 社員として、有給休暇・半休・病気休暇を申請したい。

- 受け入れ基準:
  - 休暇種類（有給休暇 / 半休 / 病気休暇）を選択できる
  - 日付を指定して申請できる
  - 申請後、ステータスが「申請中」になる
  - 有給残日数が不足している場合はエラーになる

### US-7: 休暇承認

> 管理者として、社員の休暇申請を承認または却下したい。

- 受け入れ基準:
  - 申請中の休暇一覧が表示される
  - 承認すると休暇が確定し、有給残日数が減算される
  - 却下すると申請者に結果が反映される

### US-8: 有給付与（管理者）

> 管理者として、社員に有給休暇を付与したい。

- 受け入れ基準:
  - 社員を選択し、付与日数を指定できる
  - 付与後、社員の有給残日数に反映される

### US-9: 月次勤怠集計

> 社員として、自分の月次勤怠をグラフで確認したい。

- 受け入れ基準:
  - 出勤日数・欠勤日数・有給残日数がグラフ形式で表示される
  - 月を選択して切り替えられる

### US-10: CSV エクスポート

> 管理者として、勤怠データを CSV でエクスポートしたい。

- 受け入れ基準:
  - 月・社員（全員 or 個別）を指定してエクスポートできる
  - 出力項目: 社員名、日付、出勤時刻、退勤時刻

## 勤務ルール

- 所定労働時間: 9:00〜18:00（休憩1時間含む）
- 残業計算: MVP 対象外
- 深夜割増: MVP 対象外
- 休日出勤: MVP 対象外
- 同日複数回出勤（分割勤務）: なし

## 休暇種類

| 種類 | 説明 |
|------|------|
| 有給休暇 | 管理者が付与。残日数を管理する |
| 半休 | 午前 or 午後の半日休暇。有給から 0.5 日消化 |
| 病気休暇 | 有給とは別枠（残日数管理なし） |

## MVP 対象外（将来検討）

- 残業時間の自動計算
- 深夜勤務割増計算
- 休日出勤・振替休日・代休管理
- 36協定の警告通知
- 打刻修正申請フロー
- スマートフォン対応
- 給与システム連携

## 非機能要件

- 対象規模: 約50名
- クライアント: PC ブラウザ（レスポンシブ不要）
- 技術スタック: Java / Spring Boot（Backend）、TypeScript / Next.js（Frontend）
